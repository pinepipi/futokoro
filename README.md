# ふところ.com

この出費、今して大丈夫？ 現金・収入・支出の3つを入れるだけで、購入後に生活防衛資金（生活費の何ヶ月分）が残るか、これからの12ヶ月でいちばん現金が薄くなる月がどこかを試算する、静的1ページのツールです。

**入力値はブラウザ内だけで計算し、保存も送信もしません。** サーバー送信なし・Cookieなし・localStorageなし・ログイン不要。

公開URL: https://futokoro.app

## 使い方

`index.html` をブラウザで開くだけで動きます（ビルド不要）。

1. いまの現金・預金、毎月の手取り、毎月の生活費を入れる
2. 現金が生活費の何ヶ月分あるか／3〜6ヶ月分まであといくらかが出る
3. 任意でカード一括・車検・ボーナスなどを入れると、12ヶ月の残高推移と「いちばん薄い月」が分かる
4. 「購入前チェック」に値段と購入月を入れると、買った後に3ヶ月分の備えが残るかを試算できる

## 開発

```sh
npm install
npm run build          # 公開ファイルだけを dist/ に出力
node --test domain.test.js   # 計算ロジックの単体テスト
npm run test:publish   # dist/ に公開対象だけが入ることを検証
npm run quality:gate   # ビルド + 単体 + Playwright(導線/視覚/a11y)
```

ホスティングの publish ディレクトリは `dist/` を指定してください（リポジトリ直下をそのまま公開しない構成です）。

## 構成

- `index.html` / `styles.css` / `app.js` — 画面とブラウザ内計算
- `domain.js` / `domain.test.js` — 生活防衛資金の計算ロジックとテスト
- `ads-config.js` / `ads.js` — 広告はデフォルト無効。設定で有効化するまで外部タグを読み込まない
- `_headers` — Cloudflare Pages 等で使う公開時セキュリティヘッダー
- `privacy.html` — プライバシーポリシー
- `scripts/build-public.js` — 公開ファイルだけを `dist/` にコピーするビルド
- `tests/` — E2E / 公開構成 / UI品質テスト

## データ方針

- 入力値はブラウザのメモリ上だけで扱い、再読み込みで消えます
- サーバー送信・localStorage・Cookie・アクセス解析・外部API は使いません
- 広告はデフォルト無効。有効化時のみ広告であることを明示します
